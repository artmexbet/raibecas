# Сохранение истории чатов в Redis

## Описание

Реализована функциональность сохранения истории чатов в Redis с поддержкой:
- **Сохранение полных сообщений** от пользователя и ассистента
- **Обработка чанков** (фрагментов) ответов от нейромодуля
- **Единая история чата** для каждого пользователя
- **TTL (Time To Live)** для автоматического удаления данных
- **Очистка истории чата** через HTTP endpoint

## Изменения

### 1. Конфигурация (config.go)

Добавлены новые параметры в структуру Redis:

```go
type Redis struct {
	Host       string `yaml:"host" env:"HOST" env-default:"localhost"`
	Port       string `yaml:"port" env:"PORT" env-default:"6379"`
	DB         int    `yaml:"db" env:"DB" env-default:"0"`
	ChatTTL    int    `yaml:"chat_ttl" env:"CHAT_TTL" env-default:"86400"` // 24 часа (секунды)
	MessageTTL int    `yaml:"message_ttl" env:"MESSAGE_TTL" env-default:"86400"`
}
```

**Переменные окружения:**
- `REDIS_CHAT_TTL` - время жизни для полной истории чата (по умолчанию 86400 сек = 24 часа)
- `REDIS_MESSAGE_TTL` - время жизни для временного буфера сообщений

### 2. Redis Store (redis/redis.go)

Расширены методы хранилища:

#### SaveMessage(ctx, userID, message)
Сохраняет сообщение в историю чата с установкой TTL:
- Получает существующую историю
- Добавляет новое сообщение
- Сохраняет обновленный список в Redis с TTL

#### AppendMessageChunk(ctx, userID, content, done)
Обрабатывает фрагменты ответов от нейромодуля:
- При `done=false`: накапливает чанки во временном буфере
- При `done=true`: возвращает полное сообщение и очищает буфер
- Возвращает полное сообщение или текущий фрагмент

#### ClearChatHistory(ctx, userID)
Удаляет всю историю чата для пользователя:
- Очищает основную историю
- Очищает временный буфер

#### GetChatSize(ctx, userID)
Возвращает количество сообщений в чате пользователя

### 3. Сервис чата (service/chat.go)

Обновлен интерфейс `iChatHistoryStore` для поддержки новых методов:
- `SaveMessage()` - для сохранения полных сообщений
- `AppendMessageChunk()` - для обработки фрагментов
- `ClearChatHistory()` - для очистки истории
- `GetChatSize()` - для получения размера чата

**Основная логика ProcessInput():**
1. Получает историю чата из Redis
2. Сохраняет сообщение пользователя
3. Получает ответ от нейромодуля чанками
4. На каждом чанке:
   - Накапливает контент
   - Сохраняет чанк во временный буфер
   - При завершении (done=true) сохраняет полное сообщение

### 4. HTTP Handler (handler/http/handler.go)

Добавлены:
- Обновленный интерфейс `iService` с методом `ClearUserChat()`
- Новый endpoint для очистки чата: `DELETE /api/v1/chat/:userID`

**Endpoint очистки чата:**
```
DELETE /api/v1/chat/{userID}

Response (200):
{
  "message": "chat history cleared"
}

Response (400):
{
  "error": "userID is required"
}

Response (500):
{
  "error": "could not clear chat history"
}
```

## Ключи Redis

Для каждого пользователя создаются два ключа:

1. **`chat:history:{userID}`** - полная история чата
   - Хранит массив объектов `Message` в JSON
   - TTL: `REDIS_CHAT_TTL` (по умолчанию 24 часа)

2. **`chat:temp_msg:{userID}`** - временный буфер для чанков
   - Хранит текущий накапливаемый ответ
   - TTL: `REDIS_MESSAGE_TTL` (по умолчанию 24 часа)
   - Удаляется автоматически при завершении сообщения

## Пример использования

### Отправка сообщения и получение чата

```bash
# Отправить сообщение
curl -X POST http://localhost:8080/api/v1/chat \
  -H "Content-Type: application/json" \
  -d '{
    "user_id": "user123",
    "input": "Что такое AI?"
  }'
```

### Очистка истории чата

```bash
# Очистить всю историю чата пользователя
curl -X DELETE http://localhost:8080/api/v1/chat/user123
```

## Особенности

✅ **Автоматическое удаление** - данные удаляются из Redis автоматически после истечения TTL
✅ **Потоковая обработка** - чанки обрабатываются по мере их поступления
✅ **Единая история** - один чат на пользователя, все сообщения сохраняются
✅ **Отказоустойчивость** - если Redis недоступен, логируется предупреждение, но обработка продолжается
✅ **Управление историей** - возможность очистить всю историю через API

## Поток обработки

```
Пользователь отправляет сообщение
           ↓
Сохранить user message в Redis
           ↓
Получить embeddings от нейромодуля
           ↓
Получить документы из Qdrant
           ↓
Получить историю из Redis
           ↓
Отправить запрос к нейромодулю (chat/ollama)
           ↓
Получить ответ чанками:
  • Сохранить чанк в temp_msg
  • Отправить client (streaming)
  • Если done=true: сохранить полное сообщение в history
           ↓
Завершено
```

## Обработка ошибок

При возникновении ошибок работа продолжается:
- Ошибка получения истории → пустая история
- Ошибка сохранения → логируется предупреждение
- Ошибка сохранения чанка → логируется предупреждение

Это гарантирует, что сбои в Redis не блокируют обработку чатов.
