# Управление пользователями и заявками на регистрацию

## Новые возможности

### 1. Обработка заявок на регистрацию

При клике на заявку открывается единое модальное окно для обработки заявки с выбором роли:

**Доступные роли:**

- **Пользователь (User)** - базовый доступ:
  - Только просмотр документов

- **Администратор (Admin)** - базовые права доступа:
  - Просмотр и управление документами
  - Просмотр пользователей
  - Обработка заявок на регистрацию
  - Просмотр статистики

- **Суперадминистратор (SuperAdmin)** - полные права доступа:
  - Все возможности администратора
  - Управление пользователями (редактирование, активация/деактивация)
  - Управление настройками системы

#### Использование:
1. На странице "Заявки на регистрацию" найдите нужную заявку
2. Кликните на строку с заявкой (только для заявок со статусом "На рассмотрении")
3. В открывшемся модальном окне:
   - Просмотрите информацию о пользователе (имя, email, статус, дата создания)
   - Выберите роль для пользователя
   - Нажмите "Одобрить" для одобрения с выбранной ролью
   - Или нажмите "Отклонить" для отклонения заявки (с подтверждением)
4. Также можно нажать на иконки в колонке "Действия":
   - Зеленый плюс (✓) - одобрить (откроет модальное окно)
   - Красный крест (✗) - отклонить (откроет модальное окно)

**Преимущества нового подхода:**
- ✅ Одно модальное окно вместо двух
- ✅ Вся информация и действия в одном месте
- ✅ Быстрая обработка заявок
- ✅ Выбор роли по умолчанию (User)

### 2. Редактирование пользователей

Теперь доступно редактирование данных существующих пользователей:

#### Редактируемые поля:
- Имя пользователя (username)
- Полное имя (full_name)
- Email
- Роль (User/Admin/SuperAdmin)
- Статус активности (активен/неактивен)

#### Использование:
1. На странице "Пользователи" найдите нужного пользователя
2. Нажмите на иконку "Редактировать" (синий карандаш)
3. В открывшемся модальном окне измените необ��одимые поля
4. Нажмите "Сохранить"

## Роли и регистр

⚠️ **ВАЖНО**: Все роли использу��т PascalCase регистр:
- `User` (не `user`)
- `Admin` (не `admin`)
- `SuperAdmin` (не `super_admin` или `superadmin`)

Это критично для правильной работы системы на уровне базы данных и backend.

## Технические детали

### Backend API

#### Одобрение заявки с ролью
```
POST /api/v1/registration-requests/:id/approve
Authorization: Bearer <token>
Content-Type: application/json

{
  "role": "User" | "Admin" | "SuperAdmin"
}
```

**Роль по умолчанию:** `User`

#### Отклонение заявки
```
POST /api/v1/registration-requests/:id/reject
Authorization: Bearer <token>
Content-Type: application/json

{
  "reason": "string (optional)"
}
```

#### Обновление пользователя
```
PATCH /api/v1/users/:id
Authorization: Bearer <token>
Content-Type: application/json

{
  "username": "string",
  "full_name": "string",
  "email": "string",
  "role": "User" | "Admin" | "SuperAdmin",
  "is_active": boolean
}
```

### Компоненты

#### RoleSelectionModal (обновлен)
Объединенное модальное окно для обработки заявок:
- Отображение информации о заявке (имя, email, статус, дата)
- Выбор роли с описанием прав
- Кнопки: "Одобрить", "Отклонить", "Отмена"
- Подтверждение при отклонении

**Props:**
- `visible: boolean` - видимость модального окна
- `request: CreateAdminRequest | null` - данные заявки
- `onCancel: () => void` - закрытие без действий
- `onApprove: (requestId: string, role: AdminRole) => Promise<void>` - одобрение с ролью
- `onReject: (requestId: string) => Promise<void>` - отклонение заявки

#### UserEditModal
Модальное окно для редактирования пользователя с валидацией полей.

### Миграции базы данных

Роли в базе данных используют enum типы с правильным регистром:
```sql
CREATE TYPE role_enum AS ENUM ('User', 'Admin', 'SuperAdmin');
ALTER TABLE users ALTER COLUMN role SET DEFAULT 'User'::role_enum;
```

**Важно:** Миграции уже содержат правильные значения ролей.

## Права доступа

### Матрица прав по ролям:

| Действие | User | Admin | SuperAdmin |
|----------|------|-------|------------|
| Просмотр документов | ✅ | ✅ | ✅ |
| Создание/редактирование документов | ❌ | ✅ | ✅ |
| Удаление документов | ❌ | ✅ | ✅ |
| Просмотр пользователей | ❌ | ✅ | ✅ |
| Редактирование пользователей | ❌ | ❌ | ✅ |
| Просмотр заявок | ❌ | ✅ | ✅ |
| Обработка заявок | ❌ | ✅ | ✅ |
| Просмотр статистики | ❌ | ✅ | ✅ |
| Управление настройками | ❌ | ❌ | ✅ |

Права проверяются на уровне:
- Frontend (показ/скрытие UI элементов через PermissionGuard)
- Backend (проверка прав при выполнении операций)
