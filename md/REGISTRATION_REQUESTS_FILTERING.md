# Фильтрация и улучшение страницы заявок на регистрацию

## Обзор изменений

Реализована полноценная фильтрация по статусу на странице заявок на регистрацию в админ-панели, а также приведено соответствие между фронтенд и бэкенд данными.

## Что было изменено

### Фронтенд

#### 1. Типы данных (`auth.ts`)
- Обновлен интерфейс `CreateAdminRequest`:
  - Добавлено поле `status` типа `'pending' | 'approved' | 'rejected'`
  - Добавлено поле `metadata` для дополнительных данных
  - Добавлены поля дат: `createdAt`, `updatedAt`
  - Добавлены поля одобрения: `approvedBy`, `approvedAt`
  - Удалено поле `request` (заменено на `status`)

#### 2. Сервис пользователей (`users.service.ts`)
- Обновлен метод `fetchRegistrationRequests()`:
  - Добавлена поддержка параметров фильтрации по статусу
  - Добавлена поддержка пагинации (page, page_size)
  - Используется GET запрос с query параметрами
  - Теперь возвращает объект с `requests`, `total_count`, `page`, `page_size`

#### 3. Страница заявок на регистрацию (`UserRequestsListPage.tsx`)
- **Добавлена фильтрация по статусу**:
  - Компонент Select для выбора статуса (pending, approved, rejected)
  - Возможность очистить фильтр (показать все заявки)
  - Фильтр срабатывает при изменении и сбрасывает пагинацию на первую страницу

- **Улучшена таблица**:
  - Добавлена колонка "Статус" с цветными тегами
  - Добавлена колонка "Дата создания" с сортировкой
  - Удалена колонка "Запрос" (теперь используется статус)
  - Улучшена обработка кликов на строку (клик работает только для ожидающих заявок)

- **Добавлена пагинация**:
  - Полная поддержка серверной пагинации
  - Изменяемый размер страницы (10, 20, 50 элементов)
  - Отображение общего количества заявок

- **Улучшена обработка ошибок и состояний**:
  - Сообщения об ошибках через message.error()
  - Пустое состояние (Empty компонент) когда нет данных
  - Асинхронная обработка одобрения и отклонения с перезагрузкой списка

- **Визуальные улучшения**:
  - Цветные теги для статусов (синий - в ожидании, зеленый - одобрено, красный - отклонено)
  - Кнопки "Действия" отключены для уже обработанных заявок
  - Лучшее расположение и стили элементов управления

#### 4. Моки данные (`registrationRequests.ts`)
- Обновлен интерфейс `RegistrationRequest`:
  - Удалены поля `fullName`, `reason`, `reviewedAt`, `reviewedBy`
  - Добавлено поле `metadata` для хранения дополнительных данных
  - Добавлены поля `createdAt`, `updatedAt`, `approvedAt`, `approvedBy`

- Обновлены моковые данные в соответствии с новой структурой
- Поля fullName и reason теперь хранятся в metadata

#### 5. Мок обработчики (`registrationRequests.ts`)
- Обновлен метод `getAll()`:
  - Теперь принимает параметры фильтрации и пагинации
  - Возвращает полный объект ответа с метаданными пагинации
  - Поддерживает фильтрацию по статусу

- Обновлены методы `approve()` и `reject()`:
  - Используют новые названия полей (approvedAt, approvedBy)
  - Правильно обновляют updatedAt и approvedAt

### Бэкенд

Бэкенд уже поддерживал все необходимые функции:

## API контракт

### Запрос на список заявок (GET)
```
GET /api/v1/registration-requests?page=1&page_size=10&status=pending
```

Query параметры:
- `page` (int, required) - номер страницы, по умолчанию 1
- `page_size` (int, required) - размер страницы, по умолчанию 10
- `status` (string, optional) - фильтр по статусу: pending, approved или rejected

### Ответ на список заявок
```json
{
  "requests": [
    {
      "id": "uuid",
      "username": "string",
      "email": "string",
      "status": "pending|approved|rejected",
      "metadata": {},
      "createdAt": "2026-02-03T10:30:00Z",
      "updatedAt": "2026-02-03T10:30:00Z",
      "approvedBy": "uuid (optional)",
      "approvedAt": "2026-02-03T10:30:00Z (optional)"
    }
  ],
  "totalCount": 10,
  "page": 1,
  "pageSize": 10
}
```

## Как использовать

### Фильтрация по статусу
1. На странице заявок выберите нужный статус из dropdown "Фильтр по статусу"
2. Таблица автоматически обновится, показав только заявки с выбранным статусом
3. Нажмите на "X" чтобы очистить фильтр и увидеть все заявки

### Пагинация
1. Используйте стрелки в конце таблицы для навигации по страницам
2. Выберите размер страницы (10, 20, 50) в dropdown "Entries per page"
3. Выбранный размер сохраняется при навигации

### Одобрение/Отклонение
1. Клик на заявку откроет диалог (только для заявок со статусом "pending")
2. Нажмите "Одобрить" для одобрения или "Отклонить" для отклонения
3. После успешной обработки таблица автоматически обновится

## Технические детали

### Константы статусов
- `pending` - Заявка на рассмотрении
- `approved` - Заявка одобрена
- `rejected` - Заявка отклонена

### Цвета тегов
- Синий (blue) - pending
- Зеленый (green) - approved
- Красный (red) - rejected
